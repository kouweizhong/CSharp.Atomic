Detailed Description

This interface represents a thread access locking implementation.

The common use case for these objects will in a try/finally block:

        ILock _lock;
        List<int> _integers = new List<int>();

        void AddToList(int integer) {
            _lock.Lock();
            try { 
                _integers.Add(integer);
            }
            finally {
                _lock.Unlock();
            }
        }

ICondition CSharp.Locking.ILock.NewCondition ( )

Returns an ICondition instance for use with this ILock instance.

The returned ICondition instance supports the same usages as do the Monitor methods (Monitor.Wait() and Monitor.PulseAll()).

  • If this lock is not held when any of the ICondition.Await() or ICondition.Signal() methods are called, then an exception is thrown.
  • When the condition ICondition.Await() methods are called the lock is released and, before they return, the lock is reacquired and the lock hold count restored to what it was when the method was called.
  • The ordering of lock reacquisition for threads returning from waiting methods is the same as for threads initially acquiring the lock, which is in the default case not specified.

Returns:
The ICondition object
